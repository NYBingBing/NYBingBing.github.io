iOS静态库制作

Posted on 2016-06-03

  1. 制作静态库：
    Xcode————>New————>project————>iOS——————>Cocoa Touch Static Libary

  2. 新建成功后，自动生成与文件名字相同的.h和.m文件(也可以删除之后新建，作者采用的是删除新建的方式)， 在Products文件夹中包含一个lib+项目名的.a文件（此时.a文件为红色）

  3. 添加.h文件到Copy Files中，步骤：TARGETS————>Build Phsases————>Copy Files 点击下方“+”

  4. 打包支持模拟器和真机的静态库（分别选择模拟器和真机运行，就会生成对应的静态库）
  5. 如果需要生成支持不同架构的静态库，请到 PROJECT————>Build Settings——————>Architectures————>Build Active Architecture Only ————>Debug改为NO(此处根据需要制作Debug版本或Release版本)
  6. 选中模拟器或真机，编译运行，此时.a文件会变成黑色。此时.a文件和.h 文件就可直接拖入需要的项目使用。
  7. 合并版本：（合并出适用于模拟器和真机的.a文件）：a.将文件 show in finder ,会看到对应的静态库。b.在终端中进入对应文件，输入 lipo -create SQY/iOS/iphoneos/libGamePlusAPI.a SQY/iOS/iphonesimulator/libGamePlusAPI.a -output SQY/iOS/libGamePus.a
    其中SQY/iOS/iphoneos/libGamePlusAPI.a //为真机库。
    SQY/iOS/iphonesimulator/libGamePlusAPI.a //为模拟器库
    -output SQY/iOS/libGamePus.a //为两个合并后存放的路径
    c.然后可以输入命令测试下是否成功
    lipo -info SQY/iOS/libGamePus.a //下面是输出 armv7 i386 有了两个就表情模拟器和真机都支持 其中armv7为真机架构 i386为模拟器
    Architectures in the fat file: SQY/iOS/libGamePus.a are: armv7 i386
    d.此时生成的.a文件，即为支持模拟器和真机的.a文件。
